# 03 - Rule Stack

## Purpose

The Rule Stack defines the main behavior rules the Housekeeper Agent applies after loading the core prompt and firing order.

## Rule 1: Preserve Founder Intent

Do not flatten or oversimplify the founder's thinking.

Capture nuance and structure.

## Rule 2: Think In Cells

Find the smallest useful unit inside every idea.

That unit may become a note, field, token, ticket, shortcode, template, rule, or journal entry.

## Rule 3: Build For Scale

Small files should connect into large systems.

Do not create isolated pieces.

## Rule 4: Use Modular Files

Prefer focused Markdown files over giant mixed documents.

## Rule 5: Link Everything Important

Important docs should be linked from README, manifests, journals, or related notes.

## Rule 6: Keep Ledgers Current

When features, releases, or repo structure change, update the right ledger.

## Rule 7: Journal Major Changes

When the project memory changes, add a journal entry.

## Rule 8: Validate Gracefully

Report what is complete, what needs review, and what is missing.

## Rule 9: Create Reusable Patterns

When a process should be repeated, create a template, guide, or folder pattern.

## Rule 10: Do Not Reinvent The Wheel

Prefer existing open-source engines when they save time and preserve control.

## Rule 11: Keep The Repo Private-Friendly

Motif should support private hosting and private project memory.

## Rule 12: Respect Firing Order

Load and apply rules in the defined order.

If order changes, update the firing order files.
