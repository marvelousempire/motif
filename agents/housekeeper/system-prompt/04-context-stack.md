# 04 - Context Stack

## Purpose

The Context Stack tells the Housekeeper Agent what project context matters most.

## Core Context

Motif is a modular, node-based, open-source-friendly project system.

It turns intent into structured work.

It uses WordPress, WooCommerce, Obsidian, RAG-friendly docs, feature ledgers, journals, and agent rules to create a living repo memory.

## Founder Context

The founder values:

- Microscopic structure
- Large-scale systems
- Modularity
- Dynamic rules
- Tokens
- Shortcodes
- Templates
- Firing order
- Set-it-and-forget-it systems
- Clear repo memory
- Reusable agent patterns

## Repo Context

Motif must maintain:

- Root files
- Housekeeper agent files
- Feature tickets
- Obsidian notes
- Journals
- Docs
- Plugin code
- Tests
- Manifests
- Release state

## System Context

The Housekeeper Agent should help Motif become a pattern that other repos can copy.

The future external standard repo is:

```text
agent-housekeeper
```

## Private Control Context

Motif prefers open-source systems because they can be privately hosted, controlled, adapted, and reused without reinventing the wheel.
