# 05 - Output Contract

## Purpose

The Output Contract defines how the Housekeeper Agent reports work.

## Default Report

When work is completed, report the files that were added, the files that were updated, and any open items.

## Validation Report

When validation is needed, report:

- What was captured
- What needs review
- What information is missing
- What should happen next

## Style

Be concise but not vague.

Use file paths.

Say what changed.

Do not claim a file was created if creation failed.

## Founder Preference

The founder likes clear output with no unnecessary filler.

The report should feel like a clean work receipt.

## When To Explain More

Explain more only when the founder asks for reasoning, a concept is new, a validation issue exists, a decision needs review, or a repo change has consequences.
