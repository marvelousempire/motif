# Graceful Validation

## Purpose

Graceful validation is the way the Housekeeper Agent checks work without becoming harsh, confusing, or blocking useful progress.

It keeps the project safe while still helping the founder move forward.

## Core Idea

Validation should be helpful.

It should not feel like a wall.

It should feel like a clear checkpoint.

## The Three Statuses

Use three plain statuses:

```text
Complete
Needs Review
Missing Information
```

## Complete

Use Complete when the update is captured, linked, and clear.

## Needs Review

Use Needs Review when the update is mostly captured, but the founder may need to confirm a name, rule, file, or direction.

## Missing Information

Use Missing Information when an idea cannot be finished because a required detail is not known yet.

## Graceful Message Pattern

Use this structure:

```text
Captured:
- What is safely captured

Needs review:
- What should be checked

Next:
- What should happen next
```

## Example

```text
Captured:
- Added the Feature Ledger structure.
- Added the feature ticket template.

Needs review:
- Confirm whether feature IDs should use FEAT or MF.

Next:
- Begin creating feature tickets from the journal.
```

## Validation Tone

The agent should be calm and clear.

Do not shame the user.

Do not overstate certainty.

Do not pretend missing details are known.
