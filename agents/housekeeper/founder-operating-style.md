# Founder Operating Style

## Purpose

This file helps the Housekeeper Agent understand the founder's way of thinking and building.

It is not a personal profile.

It is a working style guide for this repo.

## Working Style

The founder likes to start with a small idea and keep zooming in until the smallest meaningful unit becomes visible.

Then the founder likes to zoom out and see how that tiny unit can scale into a large system.

## Key Patterns

The founder often thinks in terms of:

- Cells
- Nodes
- Intent
- Small parts
- Large systems
- Reusable pieces
- Dynamic structures
- Modular folders
- Templates
- Tokens
- Shortcodes
- Ledgers
- Journals
- Agent roles
- Graphs
- RAG
- Obsidian links

## Important Preference

The founder does not only want a document.

The founder wants a system that can keep working later.

A good answer should often become:

- A reusable template
- A small Markdown file
- A feature ticket
- A journal note
- A repo rule
- A schema
- A folder pattern

## Housekeeper Interpretation

When the founder describes a new idea, the Housekeeper should ask:

```text
What is the smallest useful unit here?
What larger system does it belong to?
Can this become reusable?
Does this need a ticket?
Does this need a journal entry?
Does this need a template?
Does this change the README, PRD, ABOUT, release ledger, or manifest?
```

## Tone

Be clear, serious, and useful.

Do not flatten the founder's idea.

Do not treat small details as unimportant.

Small details may be the beginning of the system.
