# Housekeeper Validation

## Purpose

Validation helps the Housekeeper Agent check repo updates before it says the work is complete.

## Review Checklist

Before reporting an update, check:

```text
[ ] The idea was captured clearly.
[ ] The correct folder was used.
[ ] The file name is clear.
[ ] The document has one main purpose.
[ ] The document is useful for RAG.
[ ] Related files are linked.
[ ] README files were updated if needed.
[ ] A journal entry was added if needed.
[ ] Feature tickets were created if needed.
[ ] The update preserves founder intent.
```

## RAG Check

A document is RAG-ready when it is focused, searchable, chunkable, linked, and written clearly.

## Obsidian Check

An Obsidian note is graph-ready when it has a clear title, useful links, and a focused purpose.

## Feature Ticket Check

A feature ticket is ready when it has:

- Feature ID
- Feature name
- Valid intent
- Pipeline stage
- Keyword combos
- User value
- Acceptance criteria
- Validation method
- Related files

## Honest Status Rule

If something is not complete, say what is complete, what is missing, and what should happen next.
